# 📚 知识库录入指南

## 🎯 录入知识的方法

### 1. 使用录入工具（推荐）

运行知识录入工具：
```bash
npm run add-knowledge
```

该工具提供4种录入方式：
- **添加文本内容** - 直接输入文本内容
- **添加单个文件** - 录入指定的文件
- **添加整个文件夹** - 批量录入文件夹中的所有文档
- **退出** - 退出录入工具

### 2. 文件格式支持

系统支持以下文件格式：
- `.txt` - 纯文本文件
- `.md` - Markdown文档
- `.json` - JSON格式文件

### 3. 文档存放位置

建议将要录入的文档放在以下位置：
```
knowledge-base/
├── technical/          # 技术文档
├── business/          # 业务文档
├── faq/              # 常见问题
└── policies/         # 政策规范
```

### 4. 文档命名规范

- 使用有意义的文件名
- 避免特殊字符
- 建议使用英文或拼音
- 示例：`ai_introduction.md`、`rag_technology.txt`

## 🔧 使用示例

### 示例1：录入技术文档
```
选择: 2 (添加单个文件)
文件路径: ./knowledge-base/technical/ai_basics.md
```

### 示例2：批量录入
```
选择: 3 (添加整个文件夹)
文件夹路径: ./knowledge-base/faq/
```

### 示例3：直接录入文本
```
选择: 1 (添加文本内容)
标题: 公司简介
分类: 企业信息
内容: 我们是一家专注于AI技术的创新公司...
END
```

## 📝 最佳实践

### 文档质量
- 内容准确、完整
- 结构清晰、有条理
- 避免冗余和重复
- 定期更新维护

### 分类管理
- 按主题分类存放
- 使用统一的分类标准
- 添加有意义的标签
- 建立文档索引

### 内容组织
- 每个文档专注单一主题
- 使用标题和子标题
- 添加关键词和摘要
- 包含实用的示例

## 🔍 验证录入结果

录入完成后，可以通过以下方式验证：

1. **启动Agent系统**
   ```bash
   npm start
   ```

2. **查看知识库状态**
   ```
   输入: 知识库
   ```

3. **测试检索功能**
   ```
   输入: 什么是Agent系统？
   ```

## ⚠️ 注意事项

- 确保RAG功能已启用（RAG_ENABLED=true）
- 文档内容避免包含敏感信息
- 大文件会自动分块处理
- 录入过程中请保持网络连接稳定

## 🆘 常见问题

**Q: 录入失败怎么办？**
A: 检查文件路径是否正确，文件格式是否支持

**Q: 如何删除已录入的知识？**
A: 当前版本需要重启系统来清空内存中的知识库

**Q: 支持PDF文件吗？**
A: 当前版本暂不支持，未来版本会添加

**Q: 知识库有大小限制吗？**
A: 内存存储模式下建议控制在合理范围内
